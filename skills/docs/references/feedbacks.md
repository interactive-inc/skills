# feedbacks/

顧客の声をテーマ別に管理する。voiceスキルで追記する。

## slug.md

1テーマ1ファイル。関連する声をまとめる。

FrontMatter(任意):

```yaml
---
notion-page-id: {Notion Page ID}
---
```

本文:

```md
# {課題テーマ名}

{声の要点を文章で記述} [^1]。{別の声があれば続けて記述} [^2]。

## 課題

- {分析した課題1}
- {分析した課題2}

---

[^1]: 「{原文}」({誰}, {YYYY-MM-DD}, {ソース})
[^2]: 「{原文}」({誰}, {YYYY-MM-DD}, {ソース})
```

声は箇条書きではなく文章で記述する。脚注の前に `---` を入れる。脚注に原文・日付・ソースをまとめる。

## index.md

`signals-index` スキルで再生成する。カテゴリ別ナラティブ＋脚注形式。signal / voice スキルでは触らない。

冒頭にサマリ段落、続いてカテゴリ別の見出しとナラティブ本文、末尾に脚注を並べる。カテゴリは事前に固定せず、slug.md の内容から product ごとに自然に立ち上がる分類を使う。

```md
# フィードバック

{冒頭サマリ段落。全体傾向・注目テーマ・ビジョンとの整合}

## {カテゴリ名}

{そのカテゴリに属する複数テーマをナラティブで記述} [^1]。{別テーマの記述} [^2]。

## {カテゴリ名}

{本文} [^3]。

---

[^1]: {slug}.md ― {テーマ要約}（{N}件）

[^2]: {slug}.md ― {テーマ要約}（{N}件）
```

ルール:

- カテゴリ数は5から10程度を目安にする
- 脚注番号は本文の出現順で付ける
- 声数の多いテーマを優先して記述の粗密を調整する
- ビジョン外のテーマは本文でその旨を明示する

## ルール

- 声の原文を残す。要約しない
- 新しい声は既存テーマに追記。類似テーマがなければ新規slug.md作成
- ビジョン外の要望には `## 課題` にその旨を明記
- テーマの課題がindex.mdの「解決する問題」に昇格したら `## 課題` に明記
