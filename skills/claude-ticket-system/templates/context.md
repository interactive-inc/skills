# Context template

Written at ticket creation. Describes the background, motivation, and requirements. Technical details belong in the Issue section.

Start with a plain text summary. Add `##` headings as needed.

Common headings:

- `## Requirements` - What must be achieved.
- `## Constraints` - Limitations and boundaries.
- `## Scope` - What is and is not included.

Additional `##` headings may be added freely.
